Active Link Indicator Navigation

This navbar features a dynamic active link indicator that smoothly moves between navigation items. The indicator shows which section you're currently viewing and animates when you click different links.

Key Features:

  • Dynamic position-based indicator
  • Smooth transition animations
  • Scroll-based section highlighting
  • Real-time position calculation
  • Responsive design adaptation

About Section

This section demonstrates how the active indicator updates based on the current section in view. The JavaScript calculates which section is most visible and updates the navigation accordingly.

Technical Implementation:

The active indicator uses JavaScript to:

  • Calculate the position and width of the active link
  • Animate the indicator using CSS transforms
  • Monitor scroll position to update active states
  • Handle window resize events for responsiveness

Services Section

Notice how the active indicator smoothly transitions to this section's navigation link. The animation provides clear visual feedback about your current location in the site.

Animation Properties:

transform: translateX(position) scaleX(width);
transition: transform 0.3s cubic-bezier(0.4, 0, 0.2, 1);

The cubic-bezier timing function creates a smooth, professional feel to the animation.

Portfolio Section

The active indicator adapts to different link widths automatically. Each navigation item can have different text lengths, and the indicator will scale accordingly.

Responsive Behavior:

  • Indicator recalculates position on window resize
  • Adapts to mobile navigation layouts
  • Maintains smooth animation across breakpoints
  • Gracefully handles navigation structure changes

Testimonials Section

The scroll detection works by calculating the intersection of each section with the viewport. The section with the largest visible area becomes the active one.

Intersection Observer vs Scroll Events:

This implementation uses scroll events for educational purposes, but modern browsers support the Intersection Observer API for better performance:

  • Scroll Events: More browser support, simpler logic
  • Intersection Observer: Better performance, more complex setup

Implementation Tips:

  • Use requestAnimationFrame for smooth animations
  • Throttle scroll events to improve performance
  • Add fallbacks for browsers without transform support
  • Test thoroughly on different screen sizes
  • Consider accessibility implications of dynamic indicators
